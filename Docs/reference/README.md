# reference/ — ALWTTT

This folder contains explanatory and support material that helps readers understand the governed system.

What belongs here:
- catalogs
- explanatory summaries
- support references
- onboarding aids that do not redefine subsystem truth

What does **not** belong here:
- current subsystem authority
- current runtime authority
- active planning
- historical archive material

## Conflict rule

If a `reference/` doc conflicts with:
- `SSoT_CONTRACTS.md`, or
- an active SSoT in `systems/`, `runtime/`, or `integrations/`,

the active governed doc wins.

Reference clarifies; it does not silently redefine.

## Update rule

Update this folder when:
- a support catalog is useful for day-to-day work
- an explanatory aid improves discoverability
- a secondary doc must be aligned with an already-updated primary SSoT
