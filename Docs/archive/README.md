# archive/ — ALWTTT

This folder preserves material that is no longer primary authority but is still worth keeping for:
- traceability
- migration history
- legacy comparison
- recovery of useful ideas

What belongs here:
- superseded docs
- absorbed source docs
- historical experiments
- migration redirect maps

What does **not** belong here:
- current subsystem truth
- current runtime authority
- active planning

## Conflict rule

If an `archive/` doc conflicts with:
- `SSoT_CONTRACTS.md`, or
- an active SSoT in `systems/`, `runtime/`, or `integrations/`,

the active governed doc wins.

Archive preserves history, not authority.

## Update rule

Update this folder when:
- a previous doc is superseded
- a migration source needs a redirect target
- an absorbed source set needs a durable trace map

## Snapshot rule

The pre-governance snapshot may remain external to the active docs tree.
Use `SNAPSHOT_RETENTION_POLICY.md` and `archive/absorbed/Source_Docs_Supersession_Map.md` as the governed answer for how that snapshot should be treated.
